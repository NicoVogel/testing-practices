What is unit testing?

A practice to test parts of your code independently and quickly from other parts.

Even tough this statement might seem straight forward at first glance, there are multiple nuances to this skill and I will outline some of them.

A context

What is a context?

It can be a class or a function. But not all classes or functions count as context!

For simplicity, lets look at just a class for now.

Imagine you work on a feature and your class gets bigger and bigger. At some point you decide this is no longer understandable and you split the class into multiple classes.

The context is the main class that embeddes the other classes.

As the other classes are just implementation details

What are contracts?

A contract in computer science has:

  • pre-condition -> fulfilled before an action
  • post-condition -> fulfilled after an action
  • invariants -> always true
